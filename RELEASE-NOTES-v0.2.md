# 🚀 PMOSkills Release Notes — v0.2

We are thrilled to announce the second stable release of **PMOSkills (v0.2)** — the ultimate **operational and documentation expansion** of the industry's first executable project management framework and PMO reference architecture built directly upon **PMI PMBOK® 8th Edition** and 23 secondary companion standards.

Following the success of our v0.1 release, version **v0.2** establishes a world-class documentation layer. We have successfully authored and integrated **six brand-new master manuals** and **sustainability additions** specifically designed to turn high-level standards into active, deterministic, and highly structured procedures for both **Human Practitioners** and **Autonomous AI Agents**.

## 🌟 What's New in v0.2

### 1. 👔 Track 1 Expansion (Human Project Managers & PMO Directors)
We have authored three definitive guides to govern execution and scaling decisions:
* 📘 [**`pm-handbook.md`**](docs/human-practitioners/pm-handbook.md) — **Standard PM Operating Lifecycle Handbook.** Outlines triple-constraint control processes, Predictive execution paths, Adaptive sprint iterations, Hybrid workflows, and administrative closure SOPs.
* ✂️ [**`tailoring-guide.md`**](docs/human-practitioners/tailoring-guide.md) — **Tailoring & Scaling Parameters Guide.** Implements a strict PMBOK 8 4-step tailoring framework, scaling project decisions across **T1 (Operational) to T4 (Enterprise)** complexity. Features tailored paths for Space Tech, Green Energy, and Healthcare.
* ⚖️ [**`governance-guide.md`**](docs/human-practitioners/governance-guide.md) — **OPM & PMO Governance SOP.** Sets organizational flow mappings, PMO services catalog, master RACI frameworks, and decision threshold escalation gates.

### 🤖 2. Track 2 Expansion (AI Agents & LLM Co-Pilots)
To enable autonomous LLM orchestrators to traverse, reason, and act on this repository without human intervention, we created:
* 🧠 [**`ontology-specification.md`**](docs/ai-agents/ontology-specification.md) — **Knowledge Graph Ontology Specification.** A programmatic directed relational graph detailing edge classes (`ENFORCES`, `INPUT_TO`, `RESOLVES`), YAML front-matter specifications, and schema parsing rules.
* 🤖 [**`system-prompts.md`**](docs/ai-agents/system-prompts.md) — **System Prompts Library.** Provides ready-to-inject, fine-tuned system prompt templates for three autonomous roles: **PMO Orchestrator Controller**, **Risk and Compliance Auditor**, and **TIMWOODS Waste Performance Auditor**.

### 🛠️ 3. Track 3 Expansion (Developer Tools & Integrations)
* 🛠️ [**`automated-validators.md`**](docs/developer-tools/automated-validators.md) — **Automated Quality Gates Handbook.** Describes the L1 (Structural), L2 (Semantic), and L3 (Integration Scenario) programmatic checkers. Includes step-by-step instructions for running validators locally and setting up Git pre-commit hooks.

### 🌱 4. Track 4 Expansion (General & Sustainability Reference)
* 📈 [**`sustainability-guide.md`**](docs/general/sustainability-guide.md) — **Integrated ESG-KPI Baseline Matrix.** Appended a standard ESG mapping matrix linking specific Pack 03 Planning Skills directly to concrete environmental sustainability, circularity, diversity, and supplier auditing targets.

## 🚀 How to Get Started in v0.2

### For Human PMs and PMO Directors:
1. Open the master portal [**`docs/user-friendly-guide.md`**](docs/user-friendly-guide.md) to choose your track.
2. Review [**`docs/human-practitioners/pm-handbook.md`**](docs/human-practitioners/pm-handbook.md) for concrete lifecycle operating standards.
3. Apply tailoring scaling rules via [**`docs/human-practitioners/tailoring-guide.md`**](docs/human-practitioners/tailoring-guide.md) for your project size.

### For AI prompt developers:
1. Feed [**`docs/ai-agents/ontology-specification.md`**](docs/ai-agents/ontology-specification.md) to your LLM crawler to map the knowledge relationships.
2. Initialize agents using the prompt library in [**`docs/ai-agents/system-prompts.md`**](docs/ai-agents/system-prompts.md).
3. Validate agent generated artifacts using [**`docs/developer-tools/automated-validators.md`**](docs/developer-tools/automated-validators.md) rules.
