## Definition

The Lessons Learned Record is the continuously maintained repository of knowledge gained during the project — what worked, what did not work, and what should be done differently. It is updated throughout the project lifecycle (not only at closure) and becomes part of the Organizational Process Assets (A05) upon project close, feeding future projects.

**PMI Source:** PMBOK8 Guide §2.1.6.4 — Manage Project Knowledge; Standard §3 Principles (stewardship, learning).

## Purpose

- Capture and preserve project knowledge in a reusable, structured form
- Feed organizational process assets (A05) to improve future projects
- Provide the knowledge base for the Final Performance Report (A17 §8)
- Support retrospectives (SKL-07-03) in adaptive and hybrid projects
- Close the organizational learning loop across the project lifecycle

## Lesson Entry Schema

| Field | Description |
|-------|-------------|
| **Lesson ID** | Format: L-NNN (e.g., L-001) |
| **Date** | Date lesson was captured |
| **Phase / Context** | Lifecycle phase or event where lesson arose |
| **Category** | Scope / Schedule / Cost / Risk / Resource / Stakeholder / Process / Technical / Governance |
| **Description** | What happened |
| **Root Cause** | Why it happened |
| **Impact** | Effect on project (positive or negative) |
| **Recommendation** | What to do differently (or continue doing) on future projects |
| **Raised By** | Name and role |
| **Status** | Draft / Reviewed / Approved / Submitted to OPA |

---

## Collection Triggers

Lessons should be captured at:
- Any significant project event (positive or negative)
- Every sprint/iteration retrospective (adaptive)
- Each phase gate or milestone review
- Risk realization events
- Change request approvals (especially T3/T4)
- Issue resolution (especially escalated issues)
- Team member departure
- Project closure

## Quality Criteria

- Lessons captured throughout project — not only at closure
- Every lesson has a clear recommendation (not just a description of what happened)
- Lessons categorized and searchable by category
- Final lessons summary reviewed and approved by PM and sponsor
- OPA contribution submitted and confirmed in §5
- Adaptive projects: retrospective log updated every iteration
