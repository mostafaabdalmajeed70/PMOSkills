## Purpose

The PMBOK 8 Standard defines **six** project management principles in Section 3. These principles are not prescriptive rules — they are the foundational values and behavioral guideposts that inform how project practitioners act, decide, and lead across all projects regardless of lifecycle approach, industry, or project size.

They represent a community-driven consolidation of the twelve principles in PMBOK 7 into six broader, integrated principles. Each PMBOK 8 principle absorbs and integrates the intent of multiple PMBOK 7 principles.

## How to Use This Reference Layer

1. **For practitioners:** Use principle files to ground decisions in PMBOK 8. When facing a difficult governance or delivery judgment call, consult the relevant principle file for practitioner questions.
2. **For AI agents:** Load the relevant principle file when a skill references a principle. Principle files are the authority-grounding layer for skill execution.
3. **For auditors:** Principle files provide the traceability link from project artifacts back to PMI-stated values.
