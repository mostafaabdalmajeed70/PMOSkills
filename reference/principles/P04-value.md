# P04 — Focus on Value
**Type:** Principle
**PMBOK8 Source:** PMBOK8 Standard §3.4 — Focus on Value
**Version:** 1.0.0

## Definition
PMBOK 8 Standard §3.4 establishes that the ultimate purpose of any project is to deliver value. Value is defined as the worth, importance, or usefulness of something — and it may be financial, social, environmental, or strategic in nature. This principle requires practitioners to maintain a consistent, deliberate focus on the intended value throughout the entire project lifecycle — not only at the moment of delivery. Critically, value realization extends beyond project closure into the operational period and the broader benefits realization horizon. Projects that deliver all planned outputs on time and on budget but fail to realize the intended value have failed their fundamental purpose. Value focus also means that decisions about scope, cost, schedule, and quality should be assessed in terms of their effect on the value the project is intended to deliver.

## Key Characteristics
- **Value as the primary purpose:** All project decisions, trade-offs, and prioritizations should be traceable to the question of how they affect the value the project is intended to deliver (§3.4).
- **Multi-dimensional value:** Value encompasses financial returns, operational efficiency, strategic positioning, social outcomes, environmental impacts, and other forms of benefit — not solely monetary measures (§3.4).
- **Lifecycle-spanning focus:** Value orientation must persist from project initiation through benefits realization — it cannot be treated as relevant only at delivery or project closure (§3.4).
- **Value vs. outputs distinction:** Delivering project outputs (scope, deliverables) is necessary but not sufficient — value is realized when outputs produce the intended outcomes and benefits in the operational context (§3.4).
- **Continuous value assessment:** Practitioners must regularly assess whether the project is still positioned to deliver its intended value, and be willing to recommend course corrections — including stopping the project — when value is no longer achievable (§3.4).
- **Stakeholder-defined value:** What constitutes value must be understood from the perspective of the key stakeholders and the organization — not assumed by the project team (§3.4).

## Questions to Ask in Practice
1. Can every major project decision I am making be justified in terms of its effect on the project's intended value — or am I optimizing for delivery metrics that may be disconnected from value?
2. Is the value this project is intended to deliver still valid, realistic, and achievable given current conditions — or has the business context changed enough that the project's value case should be reassessed?
3. Do all key stakeholders share a common understanding of what value this project is intended to deliver, and how that value will be measured?
4. What will happen after this project closes — is there a clear plan and owner for capturing the intended benefits during the operational period?
5. If forced to trade off scope, cost, or schedule, which options best protect the project's ability to deliver its core value?
