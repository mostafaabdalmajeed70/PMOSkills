# PR37 — Control Quality

## Process Identity
| Field | Value |
|---|---|
| **Process ID** | PR37 |
| **Process Name** | Control Quality |
| **Process Group** | Monitoring and Controlling |
| **Knowledge Area** | Quality Management |
| **PMBOK Source** | Process Groups: A Practice Guide §7.7 |

---

## Purpose
Monitor and record results of quality management activities to assess performance and ensure project outputs are complete, correct, and meet customer expectations. Verifies that deliverables meet requirements specified by key stakeholders for final acceptance.

---

## Inputs
| Input | Description |
|---|---|
| Project management plan | Quality management plan |
| Project documents | Lessons learned register, quality metrics, test and evaluation documents |
| Approved change requests | Implemented changes to be verified |
| Deliverables | Work outputs to be inspected |
| Work performance data | Raw quality measurement data |
| Enterprise environmental factors | Industry quality standards and regulations |
| Organizational process assets | Quality checklists, historical data |

---

## Tools and Techniques
| Tool / Technique | Notes |
|---|---|
| Data gathering | Checklists, check sheets, statistical sampling, questionnaires and surveys |
| Data analysis | Performance reviews, root cause analysis |
| Inspection | Physical examination of deliverables |
| Testing/product evaluations | Functional and performance tests |
| Data representation | Cause-and-effect diagrams, control charts, histograms, scatter diagrams |
| Meetings | Quality review meetings |

---

## Outputs
| Output | Description |
|---|---|
| Quality control measurements | Measured results of quality activities |
| Verified deliverables | Deliverables confirmed correct — input to Validate Scope |
| Work performance information | Quality performance assessments |
| Change requests | Defect repair, corrective/preventive actions |
| Project management plan updates | Quality management plan |
| Project documents updates | Issue log, lessons learned register, risk register, test and evaluation documents |

---

## Key Concepts
- Quality control = **creating deliverables** that meet defined specifications (product quality).
- Quality assurance = **using project processes** effectively (process quality).
- Control Quality is performed **before** Validate Scope in most cases.
- In **agile** projects: performed by all team members throughout the project life cycle.
- In **predictive** projects: performed at specific times toward the end of the project or phase.

---

## PMBOK 8 Alignment
- Maps to **Manage Quality Assurance** (§2.1.6.5) and quality control guidance in PMBOK 8.
- Verified deliverables feed into **Validate Scope** (PR32) and **Close Project or Phase** (PR40).
