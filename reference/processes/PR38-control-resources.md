# PR38 — Control Resources

## Process Identity
| Field | Value |
|---|---|
| **Process ID** | PR38 |
| **Process Name** | Control Resources |
| **Process Group** | Monitoring and Controlling |
| **Knowledge Area** | Resource Management |
| **PMBOK Source** | Process Groups: A Practice Guide §7.8 |

---

## Purpose
Ensure that physical resources assigned and allocated to the project are available as planned, monitor planned versus actual utilization, and take corrective action as necessary.

---

## Inputs
| Input | Description |
|---|---|
| Project management plan | Resource management plan |
| Project documents | Issue log, lessons learned register, physical resource assignments, project schedule, resource breakdown structure, resource requirements, risk register |
| Work performance data | Actual resource utilization data |
| Agreements | Contract terms for external resources |
| Organizational process assets | Resource control policies |

---

## Tools and Techniques
| Tool / Technique | Notes |
|---|---|
| Data analysis | Alternatives analysis, cost-benefit analysis, performance reviews, trend analysis |
| Problem solving | Resolving resource shortages or conflicts |
| Interpersonal and team skills | Negotiation, influencing |
| Project management information system | Resource tracking systems |

---

## Outputs
| Output | Description |
|---|---|
| Work performance information | Resource utilization vs. plan |
| Change requests | Corrective action for resource shortages or surpluses |
| Project management plan updates | Resource management plan, schedule baseline, cost baseline |
| Project documents updates | Assumption log, issue log, lessons learned register, physical resource assignments, resource breakdown structure, risk register |

---

## Key Concepts
- Concerned with **physical resources** (equipment, materials, facilities, infrastructure).
- **Team members** are addressed in the Manage Team process (PR18), not here.
- Resources should be assigned and released at the **right time, right place, and right amount**.
- Changes to schedule or cost baselines require approval through PR31.

---

## PMBOK 8 Alignment
- Supported by the **Resources** performance domain (§2.5).
- Physical resource management is differentiated from team resource management in PMBOK 8.

## Practitioner Notes

Practitioners executing this process should seek answers to the following questions:
1. Are resource utilization rates monitored against the resource management plan?
2. Are team performance assessments conducted at defined intervals?
3. Are resource conflicts escalated and resolved through defined channels?
4. Is the resource breakdown structure updated to reflect actuals?
5. Are lessons learned on resource management captured during execution?
