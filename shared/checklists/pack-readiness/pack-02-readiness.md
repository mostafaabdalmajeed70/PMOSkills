## Purpose

This checklist verifies that the initiating processes (developing the charter, identifying primary stakeholders, and formulating initial business boundaries) are complete and signed off before entering the detailed planning phase (Pack 03).

---

## Readiness Checklist

| Category | Check Item | Target Skill / Reference | Verified Status |
|---|---|---|---|
| **Initiation** | [ ] Project Charter (A04) is drafted and formally approved by the Project Sponsor. | `SKL-02-01` / `A04` | [ ] Yes / [ ] No |
| **Stakeholders** | [ ] Stakeholder Register (A07) has primary internal and external stakeholders identified. | `SKL-02-02` / `A07` | [ ] Yes / [ ] No |
| **Boundaries** | [ ] Context Register (A05) establishes primary constraints (time, cost, scope bounds). | `A05` | [ ] Yes / [ ] No |
| **Prerequisite**| [ ] Pack 01 (Organizational Setup) is 100% complete and verified. | `pack-01-readiness.md` | [ ] Yes / [ ] No |

---

## Gate Transition Criteria

*   **PASS (4 / 4 met):** Project is formally authorized and stakeholders mapped. Ready to enter **Pack 03 (Planning)**.
*   **WARN (3 / 4 met):** Context Register is incomplete but Sponsor Charter is signed. **Condition:** Action plan to complete Context Register within 2 days.
*   **FAIL (< 3 met OR Charter not signed):** Formal authorization missing. Cannot enter planning phase.
