## Purpose

This checklist verifies that the necessary adaptive (Agile/Scrum) or hybrid management systems, tool configurations, backlogs, and agile ceremony cadences are established for projects using an adaptive development lifecycle.

---

## Readiness Checklist

| Category | Check Item | Target Skill / Reference | Verified Status |
|---|---|---|---|
| **Approach** | [ ] Strategic decision matrix (T1-T4) selects and justifies Adaptive/Hybrid lifecycle. | `SKL-07-01` / `A13` | [ ] Yes / [ ] No |
| **Backlog** | [ ] Scope and Requirements Decision Record WBS is structured as a dynamic backlog (A08). | `SKL-07-02` / `A08` | [ ] Yes / [ ] No |
| **Ceremonies**| [ ] Iteration cadence, planning slots, and retrospective slots (A09) are locked. | `SKL-07-03` / `A09` | [ ] Yes / [ ] No |
| **Metrics** | [ ] Velocity, burnup/burndown, and cumulative flow tracking (A17) are active. | `SKL-07-04` / `A17` | [ ] Yes / [ ] No |
| **Team** | [ ] Agile Team Operating Agreement (A09) is signed off by all team members. | `A09` | [ ] Yes / [ ] No |

---

## Gate Transition Criteria

*   **PASS (5 / 5 met):** Agile/Hybrid structures are operational. Ready for iteration execution.
*   **WARN (4 / 5 met):** Team operating agreement is draft, but backlog and ceremonies are set. **Condition:** Action plan to sign off within iteration day 1.
*   **FAIL (< 4 met OR Backlog unconfigured):** Cannot execute agile delivery. Risk of unmanaged scope chaos.
