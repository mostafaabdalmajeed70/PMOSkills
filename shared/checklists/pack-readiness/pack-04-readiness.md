## Purpose

This checklist verifies that execution requirements (resource boarding, communications protocols, procurement contracts, and team operating rules) are fully ready to support high-quality project delivery.

---

## Readiness Checklist

| Category | Check Item | Target Skill / Reference | Verified Status |
|---|---|---|---|
| **Directing** | [ ] Team Responsibility Record (A25) defines roles and RACI accountability. | `SKL-04-01` / `A25` | [ ] Yes / [ ] No |
| **Resources** | [ ] Team capacity and initial resource acquisitions (A26) are secured. | `SKL-04-04` / `A26` | [ ] Yes / [ ] No |
| **Procurement**| [ ] Sourcing strategies and vendor lists (A31/A32) are approved. | `SKL-04-05` / `A31` | [ ] Yes / [ ] No |
| **Comms** | [ ] Communications channels and reporting frequencies (A28) are configured. | `A28` | [ ] Yes / [ ] No |
| **Prerequisite**| [ ] Integrated PM Plan (A14) is 100% complete and baselined. | `pack-03-readiness.md` | [ ] Yes / [ ] No |

---

## Gate Transition Criteria

*   **PASS (5 / 5 met):** Delivery team is fully staffed, contracted, and ready to execute.
*   **WARN (4 / 5 met):** Standard vendor agreements are still in draft, but internal staff is fully onboarded. **Condition:** Action plan to execute contracts in 3 days.
*   **FAIL (< 4 met):** Critical roles unassigned or baseline plan missing. Execution cannot proceed.
