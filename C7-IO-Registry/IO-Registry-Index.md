## Purpose

This file is the **master registry of all inputs and outputs** referenced across the 40 PMBOK processes (C4 Process Records). It provides:

- A deduplicated, classified list of every artifact used as an input or output
- The artifact type (plan, document, register, baseline, report, request, data, or environmental)
- A cross-reference to every process (PR number) where it appears as input and/or output
- A frequency ranking showing which artifacts are most central to PMBOK processes

## Artifact Type Classification

| Type Code | Artifact Type | Description |
|---|---|---|
| PL | Plan / Subsidiary Plan | Management plans (components of the PMP) |
| BL | Baseline | Approved, change-controlled scope, schedule, or cost baseline |
| DOC | Project Document | Working documents updated throughout the project |
| REG | Register | Living logs that accumulate and track information |
| REP | Report | Periodic performance or status reporting artifacts |
| REQ | Request | Change requests; approved or pending |

| ENV | Environmental / Organizational Inputs | EEF and OPA — not produced by the project |
| BUS | Business Document | Business case, benefits management plan |
| EXT | External / Contractual | Agreements, procurement documents |
| OUT | Primary Deliverable Output | Tangible products or deliverables produced |

## Section 12 — Core Data Flow Chains

Key artifact chains that connect processes across Process Groups.

### Chain A — Scope Definition Chain
```
Project Charter (PR01)
  → Requirements Documentation (PR03)
    → Project Scope Statement (PR04)
      → Scope Baseline: WBS + WBS Dictionary (PR07)
        → Activity List (PR08) → Schedule (PR10) → Cost Baseline (PR13)
```

### Chain B — Work Performance Data Flow
```
Work Performance Data (PR25 — Executing)
  → Work Performance Information (PR30/31/32/34/35/37/38 — M&C)
    → Work Performance Reports (PR34 — Monitor & Control Project Work)
      → Change Requests (PR29 — Perform Integrated Change Control)
        → Approved Change Requests (PR25 — back to Executing)
```

### Chain C — Risk Chain
```
Risk Register (PR19 — Identify Risks)
  → Risk Report (PR19/PR20)
    → Qualitative Assessment (PR20)
      → Quantitative Assessment (PR21)
        → Risk Responses (PR22)
          → Risk Register updates throughout M&C (PR34, PR37, PR38)
```

### Chain D — Procurement Chain
```
Procurement Management Plan + SOW + Source Selection Criteria (PR23)
  → Bid Documents → Agreements / Contracts (PR26)
    → Control Procurements (PR38)
      → Closed Procurements → Close Project (PR40)
```

### Chain E — Knowledge & Closure Chain
```
Lessons Learned Register (captured throughout — PR27, PR34, PR35, PR36)
  → Final Report (PR40)
    → OPA Updates (PR40)
      → Future Project Input (EEF/OPA)
```
